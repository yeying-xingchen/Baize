# 白泽 Baize

白泽是一款个人 AI 助手软件，目标是让用户通过自然语言理解、管理和操作自己的电脑。它不是一个只负责问答的聊天机器人，而是面向本地文件、系统设置、应用调用与跨端协同的智能执行入口。

白泽的名称取自中国古代神兽“白泽”。传说中白泽通晓万物、知晓百怪，本项目也希望成为用户身边理解环境、辅助决策、执行任务的智能伙伴。

## 项目定位

白泽定位为操作系统层级的个人 AI 助手，核心目标是把电脑从“需要用户逐层寻找入口的工具集合”，转变为“可以被自然语言驱动的智能终端”。

用户可以用接近日常表达的方式描述目标，白泽负责理解意图、拆解任务、调用对应能力并给出可确认、可追踪、可回滚的执行过程。

## 核心愿景

- 让文件可理解：不仅按文件名查找，也能根据内容、语义、图片文字和上下文检索资料。
- 让系统可对话：通过自然语言查询配置、调整设置、诊断问题和执行维护任务。
- 让应用可调度：统一调用本地应用、网页服务和自动化流程，降低重复操作成本。
- 让任务可追踪：复杂任务拆解为可观察的步骤，关键操作需要用户确认。
- 让隐私可控制：优先考虑本地处理能力，对敏感数据和高风险操作提供明确边界。

## 主要能力规划

### 1. 智能文件管理

白泽计划围绕本地文件构建个人知识库能力，包括：

- 本地文件索引
- 文档内容理解
- 图片 OCR 识别
- 语义化文件搜索
- 文件自动分类与归档
- 重复文件识别
- 文档摘要与信息提取
- 常见格式转换

示例指令：

```text
帮我找一下上个月关于项目预算的表格。
```

```text
把桌面上的 PDF 按年份和主题整理到文档目录中，执行前先给我确认整理方案。
```

### 2. 系统助手

白泽计划提供面向个人电脑的系统辅助能力，包括：

- 查询硬件与系统信息
- 检查磁盘、内存和网络状态
- 管理开机启动项
- 清理临时文件
- 调整常用系统设置
- 诊断常见电脑问题
- 生成系统状态报告

示例指令：

```text
检查一下我的电脑为什么最近开机变慢了。
```

```text
帮我查看当前电脑配置，并判断是否适合运行大型本地模型。
```

### 3. 应用与网页自动化

白泽计划作为应用调度入口，帮助用户减少重复点击和跨应用切换：

- 启动本地应用
- 调用浏览器完成网页任务
- 提取网页信息
- 执行表单填写等自动化流程
- 将多个应用步骤编排为一个任务

示例指令：

```text
打开浏览器，搜索今天的 AI 行业新闻，并总结成要点。
```

```text
把这份会议纪要整理成待办事项，然后保存到指定文件夹。
```

### 4. 多智能体任务编排

白泽参考多 Agent 协作思路，计划将复杂任务拆分给不同能力模块处理：

- 主控 Agent：理解用户意图、拆解任务、协调执行流程
- 文件 Agent：负责文件检索、解析、整理和转换
- 系统 Agent：负责系统信息、设置和维护任务
- 应用 Agent：负责本地应用与网页自动化
- 搜索 Agent：负责联网检索、资料聚合和摘要生成
- 安全 Agent：负责权限校验、风险判断和用户确认

这种架构的目标是让白泽既能处理简单问答，也能执行跨文件、跨应用、跨系统的复杂任务。

### 5. 隐私与安全机制

白泽会把隐私和安全作为核心设计原则：

- 敏感文件默认不上传
- 高风险操作执行前必须确认
- 批量删除、覆盖、移动文件前展示计划
- 系统设置修改前展示影响范围
- 尽量保留操作记录，支持审计和回滚
- 区分普通任务、敏感任务和危险任务

高风险操作包括但不限于：

- 删除文件
- 覆盖文件
- 修改系统关键设置
- 管理启动项
- 执行脚本或命令
- 访问隐私目录
- 操作账号、资金或支付相关页面

## 适用场景

- 文件混乱，需要自动整理和检索
- 经常处理文档、表格、图片和资料
- 希望用自然语言完成电脑设置和维护
- 希望将重复办公流程自动化
- 希望构建本地个人知识库
- 希望 AI 助手在保证隐私的前提下操作本机环境

## 当前状态

项目处于早期规划阶段，当前仓库主要用于沉淀产品方向、功能设计与后续实现。具体技术栈、模块划分、安装方式和开发命令会随着代码实现逐步补充。

## 计划中的模块

```text
Baize
├── Core             # 意图理解、任务拆解与调度核心
├── Agents           # 文件、系统、应用、搜索等智能体
├── Runtime          # 本地执行环境与任务运行时
├── Knowledge        # 本地索引、语义检索与知识库
├── Security         # 权限、安全策略与操作确认
├── UI               # 桌面端或 Web 交互界面
└── Integrations     # 浏览器、系统 API、第三方服务集成
```

## 设计原则

1. 用户始终拥有最终控制权。
2. 重要操作必须可确认、可追踪、可撤销。
3. 能本地处理的敏感数据优先本地处理。
4. 不为了自动化牺牲安全边界。
5. 不把 AI 回答等同于真实执行结果，执行过程需要状态反馈。
6. 功能设计优先服务真实电脑使用场景，而不是展示性 Demo。
